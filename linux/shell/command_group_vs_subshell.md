# コマンドグループ (Command Group)
複数のコマンドを現在のシェルセッション内でグループ化して実行する。

## 構文の注意点
- 開始の`(`の後にスペースが必要。
- 最後のコマンドの後に`;`（または改行）が必須。

## 特徴
- 変数の共有：グループ内で定義・変更した変数は、終了後も現在のシェルに残る。
- オーバーヘッド：新しいプロセスを作らないため、サブシェルよりわずかに高速。

## 主な用途
- 複数のコマンド出力をまとめてリダイレクトしたいとき。
- 現在のシェルの変数状態を維持しつつ、処理をひとまとめにしたいとき。


# サブシェル (Sub Shell)
複数のコマンドを新しい子プロセス（サブシェル）を生成して実行する。

## 構文の注意点
- 前後のスペースや末尾の`;`は必須ではない（可読性のためにスペースを入れることは多い）。

## 特徴
- 変数の確率：サブシェル内での変数変更や`cd`によるディレクトリ移動ディレクトリ移動は、親シェル（元のシェル）には一切影響しない。
- 環境のコピー：親シェルの変数は引き継ぐが（エクスポート済みの場合）、書き換えても親には戻らない。

## 主な用途
- 一時的にディレクトリを用意して作業したいとき（例：`(cd /tmp && ls)`）。
- 現在の環境を汚さずに、一時的な変数を使って処理を行いたいとき。


# 比較表
| 項目 | コマンドグループ | サブシェル |
| --- | --- | --- |
| 実行プロセス | 現在のシェル | 新しい子プロセス（サブシェル） |
| 変数への影響 | あり（親シェルに残る） | なし（使い捨て） |
| ディレクトリ移動 | 親シェルのカレント位置が変わる | 親シェルには影響しない |
| 速度 | 高速（プロセス生成なし） | わずかに遅い（forkが発生） |
| 構文制御 | 厳格（スペースと`;`が必要） | 緩やか |


# 使い分けの判断基準
| やりたいこと | 推奨 | 理由 |
| --- | --- | --- |
| 複数の出力を1つのファイルへ | コマンドグループ | 効率が良く、変数を汚さない |
| 一時的な`cd`や`export` | サブシェル | 終われば環境が勝手にクリーンアップされる |
| ループ内でカウントアップ | コマンドグループ | 現在のシェルの変数に値を書き込める |
| バックグラウンド並列処理 | サブシェル | `&`と組み合わせてプロセスを分離しやすい |


# サンプルコード

## 1. リダイレクトを1つにまとめる（コマンドグループ）
ログ出力やファイル書き込みの際、複数の`echo`やコマンドの結果を一つのファイルにまとめたい場合に便利。
> ポイント：書き込み回数（システムコール）をまとめられるため、現在のプロセス内で効率的に処理できる
```bash
# 個別にリダイレクトすると面倒
echo '--- Start Log ---' >> app.log
date >> app.log
./run_backup.sh >> app.log
echo '--- End Log ---' >> app.log

# コマンドグループなら一括でリダイレクトできる
{
  echo '--- Start Log ---'
  date
  ./run_backup.sh
  echo '---End Log ---'
} >> app.log
```

## 2. 一時的なディレクトリ移動（サブシェル）
特定のディレクトリでだけコマンドを実行し、終わったら元の場所に戻りたい場合に最適。
> ポイント：スクリプト内で`cd`を多用すると「今どこにいるか」を見失いやすいが、サブシェルならその心配がない。
```bash
# サブシェルを使わない場合（cd で戻る必要がある）
cd /var/log/nginx
tar -czf ~/logs_backup.tar.gz access.log
cd -  # 元のディレクトリの戻る

# サブシェルを使う場合
(cd /var/log/nginx && tar -czf ~/logs_backup.tar.gz access.log)  # この直後、カレントは元の場所のまま
```

## 3. パイプラインでの変数保持（コマンドグループ）
少し高度だが、実務ではまりやすいポイント。
> ポイント：現在のシェルで変数を維持したい場合は、パイプではなくリダイレクトとコマンドグループを組み合わせるのが定石。
```bash
# 【失敗例】サブシェル（パイプラインの後ろはサブシェルになる）
count=0
cat list.txt | while read line; do
  count=$((count + 1))
done
echo "Total: $count"  # -> 0 と表示される（while 内は別プロセスなので親に反映されない）

# 【成功例】コマンドグループで入力を受け取る
count=0
{
  while read line; do
    count$((count + 1))
  done
  echo "Total inside: $count"  # -> 正しくカウントされる
} < list.txt
```

## 4. 並列実行と待ち合わせ（サブシェル）
バックエンドのバッチ処理などで、複数の処理を同時に走らせたい時に使用する。
> ポイント：`( ... ) &`とすることで処理を完全に切り離してバックグラウンドへ投げることができる。
```bash
# 2つの重い処理を並列で実行
( ./process_a.sh ) &
( ./process_b.sh ) &

# 両方が終わるのを待つ
wait
echo 'All processes finished'
```
