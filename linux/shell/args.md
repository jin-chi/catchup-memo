# 変数
- Linuxのコマンドライン（シェル）で扱う変数には、「シェル変数」と「環境変数」の2種類ある。
- ただ代入しただけの状態はシェル変数と呼び、「他のプログラムからも見える状態」にした変数を環境変数と呼ぶ。

## 比較表
| 項目 | シェル変数 | 環境変数 |
| --- | --- | --- |
| 定義方法 | `VAR=value` | `export VAR=value` |
| 子プロセスへの継承 | しない | する |
| 主な用途 | 一時的な変数、ループ用など | システム設定、アプリの設定 |
| 確認コマンド | `set` | `printenv`または`env` |

## TIP
- 慣習として、環境変数はすべて大文字（例：`PATH`, `HOME`）、シェル変数は小文字で書くことが多いが、システム上の強制ルールではない。
- ShellScript などで更新しない値などを大文字で記載することもよくあるので、状況によって臨機応変に使い分けるのが良い。

# シェル変数（Shell Variables）
コマンドラインで`NAME="aaa"`のように代入しただけの変数はこれにあたる。
- 有効範囲: そのシェル（ターミナル）内だけで有効。
- 特徴: そのシェルから実行した別のプログラムやスクリプトには引き継がれない。
- 用途: 一時的な作業用や、そのスクリプト内だけで使う設定などに利用される。

```bash
# ダブルクォート
var="aaa"

# シングルクォート
var='bbb'

# クォートなし (※スペースがある文字列はクォート必須)
var=ccc
```

---

# 環境変数（Environment Variables）
シェル変数を`export`コマンドで書き出ししたもの。
- 有効範囲：そのシェルと、そこから起動されたすべてのプログラム（子プロセス）で有効。
- 特徴：システム全体の設定（言語設定 LANG や実行ファイルのパス PATH など）によく使われる。
- 作り方：`export NAME="aaa"` と実行すると、環境変数になる。

```bash
# export が必要 (export がつくだけで代入ルールはシェル変数と一緒)
export NAME="aaa"
```

# 変数の指定方法
変数の指定方法は複数ある。
| 方法 | 書き方の例 | 有効範囲（スコープ） | 主な用途 |
| --- | --- | --- | --- |
| シェル変数 | `NAME=value` | 現在のシェルのみ | スクリプト内での一時保持 |
| 環境変数 (`export`) | `export NAME=value` | 現在のシェル + その後起動する子プロセス全て | システム設定、アプリの設定 |
| インライン指定 | `NAME=value command` | そのコマンド（プロセス）のみ | 一時的な言語の切り替え、デバッグ |
| 恒久設定 | `.bashrc`等に記述 | ログイン中ずっと | 基本となる言語設定、PATH の追加 |

## シェル変数
最もシンプルな代入。  
環境変数との影響範囲の違いは、`bash -c 'echo $var'`で変数を出力してみるとわかりやすい。
- 特徴：そのターミナル画面（または実行中のスクリプト）の中でしか使えない。
- 注意：`ls`, `date`などの外部コマンドを動かしても、そのコマンドの中までは値が届かない。

## 環境変数
変数を「外部に公開」するイメージ。
- 特徴：一度実行すると、そのターミナルを閉じるまで、その後実行する全てのコマンドに値が引き継がれる。
- 例：`export LANG=en_US.UTF-8`と打つと、その後のコマンドの出力が全て英語になる。

## インライン指定
その場限りの指定。
- 特徴：指定したコマンドに対してだけ、環境変数として値を渡す。
- メリット：現在のシェルの設定（`LANG`など）を汚さずに、1回だけ挙動を変えられる。

## 恒久設定（設定ファイル）
PCを再起動しても有効にしたい場合。
- 場所：`~/.bashrc`や`~/.bash_profile`など。
- 書き方：ファイルの末尾に`export LANG=en_US.UTF-8`と追記する。

# 変数の展開

## 基本的な展開
```bash
# ex
var='aaa'

# 変数の指定方法
echo $var  # -> aaa
echo "$var"  # -> aaa
echo ${var}  # -> aaa
echo "${var}"  # -> aaa

# 文字列の中で指定したい場合などは {} や ダブルクォートをよく使う
echo "set ${var}iable"  # {} が無いと variable という変数を探してしまう

# シングルクォートでは変数が展開されない
echo '$var' # -> $var
```

## デフォルト値の操作
変数が未設定、または空の場合にデフォルト値として値を設定または展開する方法。
```bash
### ※変数 var は未設定、または空が前提

# default を返す (変数は未設定、または空のまま)
echo ${var:-default}  # -> default
echo $var  # -> 未設定、または空のままなので何も出力されない

# default を返し、var にも代入
echo ${var:=default}  # -> default
echo $var  # -> default

# 値が設定済みの場合のみ、値を返す。未設定または空の場合は空を返す
echo ${var:+alternative}  # -> 何も表示されない (var が未設定、または空の場合)
echo ${var:+alternative}  # -> alternative (var がからじゃない場合)

# var が設定されているなら値を返し、そうでないなら message を返してスクリプトを終了する
: ${var:?message}  # エラーメッセージを出力して exit 1 をするのと同じ
```

## 前方一致/後方一致
変数の値から特定の部分を削除して取り出したい場合に使用する。
- 前方一致はヒットした値の前方を削除したい場合に使用するイメージ
- 後方一致はヒットした値の後方を削除したい場合に使用するイメージ

```bash
# (前方初回一致)
var="path/to/file.txt"
echo "${var#*/}"  # -> to/file.txt

# (前方最大一致)
echo "${var##*/}"  # -> file.txt
echo "${var##*.}"  # 拡張子を取得する -> txt

# (後方初回一致)
echo "${var%/*}"  # -> path/to

# (後方最大一致)
echo "${var%%/*}"  # -> path
```

## 文字列の置換・抽出
変数の値を置換したり、特定の部分だけ抽出して出力することができる。
```bash
# 最初のマッチを変換
echo ${var/aaa/bbb}

# 全てのマッチを変換
echo ${var//aa/bbb}

# 先頭が aaa なら変換
echo ${var/#aaa/bbb}

# 末尾が aaa なら変換
echo ${var/%aaa/bbb}

# 指定した文字数以降の文字列を抽出
echo ${var:3}

# 指定した文字数分抜き出す
echo ${var:0:2}

# 文字数を出力
echo ${#var}
```

## 文字数や要素数
```bash
# 文字数を取得
echo ${#var}

# 配列の要素数
echo ${#array]@]}
```

## 大文字・小文字の変換
```bash
# 最初の文字を大文字に変換
echo ${var^}

# すべての文字を大文字に変換
echo ${var^^}

# 最初の文字を小文字に変換
echo ${var,}

# すべての文字を小文字に変換
echo ${var,,}
```

# 特殊変数（特殊パラメータ）
スクリプト実行時に自動的に設定される、シェルに内蔵された特殊な変数。  
これらは基本的にはダブルクォートで囲んで使用することが推奨される。（思わぬ挙動になることがあるため）。  
例外として、`$?`, `$#`は数値が返されることが前提なので囲まない場合もあるが、囲んでも問題ない。  

- `$#`：スクリプトや関数に渡された引数の個数
- `$@`：スクリプトや関数に渡されたすべての引数（個別の文字列として展開される）
- `$*`：スクリプトや関数に渡されたすべての引数（1つの文字列として展開される）
- `$?`：直前に実行したコマンドの終了ステータス（成功は0、失敗は0以外）
- `$$`：現在実行中のスクリプトのプロセスID（PID）
- `$0`：実行されたスクリプトの名前
- `$1,$2,...`：個別の引数（位置パラメータ。何番目の引数。）
