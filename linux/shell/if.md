# if
条件判定。  
Linux 標準仕様の`test`コマンドを使用して実現する。  
`test`コマンド自体にすでに条件判定の機能が備わっており、処理内容が一行で済む場合は`if`は記載しないのが推奨される。  
`if`が必要になる場面は、条件に合致した際の処理が複数ある場合や`elif`, `else`などのその他の条件を指定したい場合。

---

# 基本
- 処理が一行の場合は、基本的には`if`は省略し、`&&`, `||`を使用して記載するのが推奨される
- 条件内の処理が複数行になったり、`else`, `elif`を使用する必要がある場合は`if`の記載方法を使用する
- 条件判定には POSIX標準の書き方と Bash で使用できるより多機能な書き方がある

## POSIX標準記法
- POSIX 標準の`test`コマンドは一重の`[]`
```bash
# 一行で書くする方法
[ "$var" = "aaa" ] && echo 'true'  # &&：終了コード 0
[ "$var" != "aaa" ] || echo 'true'  # ||：終了コード 0以外

# if を使った記載方法
if [ "$num" -gt 0 ]; then
  echo "整数です"
elif [ "$num" -lt 0 ]; then
  echo "負数です"
else
  echo "0です"
fi
```

## Bash記法
- Bash では二重の`[[]]`が使用でき、より多機能
- 変数をクォートで囲まなくても値を正しく扱ってくれる

```bash
# 一行で書く方法
[[ $var == "aaa" ]] && echo 'true'
[[ $var != "aaa" ]] || echo 'true'

# if を使った記載方法
if [[ $num -gt 0 ]]; then  # [[]] と == を使う
  echo "整数です"
elif [[ $num -lt 0 ]]; then
  echo "負数です"
else
  echo "0です"
fi
```

## 「POSIX標準」と「Bash拡張」のクォートの有無による変数の扱い
| 項目 | `[ $var ]` (`test`) | `[[ $var ]]` (Bash拡張) |
| --- | --- | --- |
| スペース | 必須（ないとエラー） | 不要（あっても安全） |
| 記号 (`*`, `?`) | 必須（展開される） | 不要（リテラルとして扱う） |
| 空変数 | 必須（消滅してエラー） | 不要（空文字として扱う） |

## コマンドの結果判定
`test`コマンドを使用するのは、文字列判定や数値判定などの記号や演算子を使用する場合。  
コマンドの結果を条件判定に使用する場合は、`[]`, `[[]]`などは不要になる。
```bash
if grep -q 'error' input.log; then
  echo 'errorが見つかった'
fi

# 一行で書くとわかりやすい
grep -q 'error' input.log && echo 'errorが見つかった'

# ×：初心者がやりがちな回りくどい書き方
if [ $(grep -q 'error' input.log) -gt 0 ]; then  # <- 動きはするがコマンド置換など無駄なプロセスが動く
  echo 'errorが見つかった'
fi
```

# 1. 文字列の比較
文字列の比較は「文字列比較演算子」を使用する。

| 演算子 | 意味 | 備考 |
| --- | --- | --- |
| `=` (`==`) | 等しい | `[ ]`では`=`が標準 |
| `!=` | 等しくない | |
| `-z` | 空である | **Z**ero (長さが0) |
| `-n` | 空ではない | **N**on-zero |

## `-n`オプション
変数が空ではないことを確認する。  
空でなければ`0`。空なら`1`を返す。
```bash
if [ -n "$var" ]; then
  echo "変数varは空ではない: ${var}"
fi

# 空の場合に処理したいなら否定をつける
if [ ! -n "$var" ]; then  # ! で否定
  echo "変数varは空です"
fi
```

## パターンマッチング（ワイルドカード）
Bash の拡張記法`[[ ]]`を使うと、SQL の`LIKE`のような「あいまい比較」ができる。
```bash
# 特定の文字を含むか
[[ $file == *.log ]] && echo 'yes'

# 特定の文字で始まるか
[[ $id == user_* ]] && echo 'yes'
```

## 正規表現マッチング（最上位の比較）
「数字だけか？」「特定のフォーマットか？」といった複雑な判定には、`=~`を使う。
- 記法：`[[ $str =~ 正規表現 ]]`
```bash
# 変数が数字のみか
if [[ $var =~ ^[0-9]+$ ]]; then
  echo "varは数字のみです: $var"
fi
```

# 2. 数値の比較
数値の比較は記号を使わず、以下の専用の演算子を使う。

| 演算子 | 意味 | 覚え方 |
| --- | --- | --- |
| `-eq` | 等しい | **Eq**ual |
| `-ne` | 等しくない | **N**ot **E**qual |
| `-gt` | より大きい | **G**reater **T**han |
| `-ge` | 以上 | **G**raeter or **E**qual |
| `-lt` | より小さい | **L**ess **T**han |
| `-le` | 以下 | **L**ess or **E**qual |

```bash
if [ "$num" -gt 0 ]; then    # 0より大きい
  echo "整数です"
elif [ "$num" -lt 0 ]; then  # 0より小さい
  echo "負数です"
else                         # 上記以外（つまり0）
  echo "0です"
fi
```

# 3. ファイルテスト演算子
運用保守のスクリプトで最もよく使われる、シェル特有の強力な演算子。

| 演算子 | 意味  |
| --- | --- |
| `-e` | 指定したパスに対象（ファイル・ディレクトリ・リンク等）が存在することを確認する |
| `-f` | 通常のファイルが存在することを確認する |
| `-d` | ディレクトリが存在することを確認する |
| `-r` | 対象が存在し、かつ現在の実行ユーザーに読み取り権限があることを確認する |
| `-s` | ファイルが存在し、かつファイルサイズが0ではない（空ではない）ことを確認する |
| `-w` | 対象が存在し、かつ現在の実行ユーザーに書き込み権限があることを確認する |
| `-x` | 対象が存在し、かつ現在の実行ユーザーに実行権限があることを確認する |
| `-L` | シンボリックリンクが存在することを確認する |

## `-e`オプション
指定したファイルやディレクトリなどが存在することを確認する。  
ファイルかディレクトリかリンクかは問わない。  
判定対象がファイルやディレクトリ、リンクなどわかっている場合は、`-f`, `-d`, `-L`を使用する方がいい。
```bash
if [ -e example.txt ]]; then
  echo 'exists'
fi
```

## `-f`, `-d`オプション
ファイルやディレクトリの存在確認をするためのオプション。  
ファイルまたはディレクトリが存在するなら`0`。存在しないなら`1`を返す。
```bash
# ファイル
if [ -f "/var/log/example.log" ]; then
  echo 'exists'
fi

# ディレクトリ
if [ ! -d "/var/log/example.log" ]; then  # ! で否定
  echo 'not exists'
fi
```

## `-s`オプション
ファイルが存在する、かつサイズが０ではないことを確認するためのオプション。  
ファイルが存在しない、または空（サイズが０）の場合は`false`を返す。  
```bash
if [ -s example.txt ]; then
  echo 'ファイルが存在する、かつ空ではない'
fi
```

# 4. 論理演算子
複数の条件を組み合わせる際に使う。  
POSIX標準では AND, OR 条件の可読性が低い。  
そのため、Bash なのであれば Bash の記法を使用することが推奨される。

| 種類 | `[ ]`(POSIX) | `[[ ]]`(Bash) | 意味・役割 |
| --- | --- | --- | --- |
| AND（かつ） | `-a` | `&&` | 左の条件（コマンド）が成功したら右を実行する |
| OR（または） | `-o` | `\|\|` | 左の条件（コマンド）が失敗したら右を実行する |
| NOT（否定） | `!` | `!` | 条件の結果を反転させる（0なら1に、0以外なら0にする） |

```bash
# POSIX標準
if [ "$var" = "aaa" -a "$status" = "active" ]; then
  echo "OK"
fi

# Bash
if [[ $var == "aaa" && $status == "active" ]]; then
  echo "OK"
fi

# どちらでも動く安全な書き方
if [ "$var" = "aaa" ] && [ "$status" = "active" ]; then
  echo "OK"
fi
```

# 応用

## 引数のバリデーション
ShellScript で受け取った引数のバリデーションをするときによく使用する方法。
```bash
# 引数の数
if [ "$#" -ne 1 ]; then  # 引数の数が1つではない場合
  echo '引数を1つだけ指定してください'
fi

# 引数のフォーマット
if [[ ! "$1" =~ ^[0-9]{4}-[0-9]{2}$ ]]; then  # ! で否定になっている
  echo '第一引数のフォーマットが不正です'
fi
```
