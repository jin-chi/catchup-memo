# ShellScript

## 関数
- いくつかの定義方法があるが、特に理由がないなら POSIX 標準の方法を使用する方がいい
- Ash, Dash では`function`は使用できない。Dash を採用している Ubuntu などでは使えない
- POSIX に準拠しているのは`func()`なので、これが推奨
```bash
# POSIX 標準
hoge() {
  ...
}

# Bash, Zsh のみ
function func {  # カッコなし
  ...
}

function hoge() {  # カッコあり
  ...
}
```

## 変数属性
```bash
# readonly をつけることで更新不可の変数にする
readonly var="$1"

# local をつけることで変数のスコープが function 内のみになる
function hoge() {
  local var="$1"
}
```

# スクリプト別のスクリプトを呼び出す際のエラーハンドリングについて
- 呼び出される側のスクリプトでexit 1を返す場合は呼び出す側のスクリプトまで処理が終了してしまうので注意
- base ./ で実行した場合は別プロセスとして実行される。リターンコードはexitで受け取る
- sourceで実行した場合はexitのリターンコードは受け取れない。return 1などで返す必要がある
- bash ./ で実行した場合は別プロセスのため別ファイルでsourceで読み込んだ環境変数などは引き継げないので、実行側でもsourceコマンドをする必要がある
