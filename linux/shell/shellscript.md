# ShellScript

## 関数
- いくつかの定義方法があるが、特に理由がないなら POSIX 標準の方法を使用する方がいい
- Ash, Dash では`function`は使用できない。Dash を採用している Ubuntu などでは使えない
- POSIX に準拠しているのは`func()`なので、これが推奨
```bash
# POSIX 標準
hoge() {
  ...
}

# Bash, Zsh のみ
function func {  # カッコなし
  ...
}

function hoge() {  # カッコあり
  ...
}
```

## 変数属性
```bash
# readonly をつけることで更新不可の変数にする
readonly var="$1"

# local をつけることで変数のスコープが function 内のみになる
function hoge() {
  local var="$1"
}
```

# エラーハンドリング

## `-euo pipefail`
基本的にはこのオプションを必須でつけるようにする。強いていうなら`-u`, `-o pipefail`は状況によってはつけなくてもいい。
- `-e`：コマンドが失敗した瞬間にスクリプトを終了するためのオプション
  - デフォルトではコマンドが失敗しても、後続の処理は続いてしまう
- `-u`：未定義、または空の変数が処理に使用された際にエラーを出す
  - デフォルトだと空のまま処理が行われる
- `-o pipeline`：パイプ連結した処理を失敗した時点で止めてエラーを出す
  - デフォルトだとパイプ連結した最後のコマンドさえ成功すればOKとみなされてしまう
- 処理の中で一時的に戻したい場合は`set +e`のように記述すればデフォルトに戻せる
- `-x`：実行しているコマンドを出力する。デバッグ時に重宝する。出力が多くなるので、正式リリース時には削除する方がいい。

```bash
# 空変数を許容したい場合
value="${1:-}"

# 変数が指定されなかった場合はデフォルト値を使いたい場合
value="${1:-default_value}"

# :? では一応改行はできる。エスケープ文字は使えない
value="${1:?
error
引数を指定してください。}"

# $# は引数の数は未定義という概念が存在しないため使える
if [ $# -eq 1 ]; then ...
```

# スクリプト別のスクリプトを呼び出す際のエラーハンドリングについて
- 呼び出される側のスクリプトでexit 1を返す場合は呼び出す側のスクリプトまで処理が終了してしまうので注意
- `base ./`で実行した場合は別プロセスとして実行される。リターンコードは`exit`で受け取る
- `source`で実行した場合は`exit`のリターンコードは受け取れない。`return 1`などで返す必要がある
- `bash ./`で実行した場合は別プロセスのため別ファイルで`source`で読み込んだ環境変数などは引き継げないので、実行側でも`source`コマンドをする必要がある
