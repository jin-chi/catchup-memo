# cron
指定した日時や定期的な間隔でコマンドやスクリプトを実行するジョブスケジューラー（常駐デーモン）。

---

# 基本
基本的な概念として、`cron`と`crontab`について押さえておく。

## `cron`(デーモン)
バックグラウンドで常時動作するプログラム。crond とも呼ばれる。

## `crontab`(テーブル)
cron が参照する設定ファイル。

---

# `crontab`の記述方法
設定ファイルには1行に1つのタスクを記述する。
```
# 分 時 日 月 曜日 コマンド
* * * * * {{コマンド}}
```
| コマンド | 内容 | 設定値 |
| --- | --- | --- |
| 分 | 何分に実行するか | `0-59` |
| 時 | 何時に実行するか | `0-23` (24時間制) |
| 日 | 何日に実行するか | `1-31` |
| 月 | 何月に実行するか | `1-12` |
| 曜日 | 何曜日に実行するか | `0-7` (0または7が日曜日) |
| コマンド | 実行するコマンド | フルパス推奨 |

## 曜日の指定と特殊文字
| 設定値 | 意味 | 例 |
| --- | --- | --- |
| `*` | 全て | 分が`*`なら毎分 |
| `,` | 複数指定 | `0,15,30,45 * * * *`で15分おき (`*/15` でも可能) |
| `-` | 範囲指定 | `0 9-15 * * *`で9時から15時まで毎時0分 |
| `/` | 間隔指定 | `*/10 * * * *`で10分おき |
| 曜日(`0-7`) | 曜日指定 | `0,7`:日曜、`1`:月曜、`2`:火曜、`3`:水曜、`4`:木曜、`5`:金曜、 6`:土曜 |
> 曜日は英字でも指定可能：SUN, MON, TUE, WED, THU, FRI, SAT

## `crontab`コマンド操作
ユーザーごとの設定は以下のコマンドで操作する。
```bash
# crontab の編集 (vi 形式で設定ファイルが開く)
crontab -e

# 設定ファイルの参照
crontab -l

# 特定のユーザーの crontab を編集 (root 権限必要)
crontab -u example-user -e
```

# 気を付けること
`cron`を安定運用するためのポイント。

## 1. コマンドはフルパスで指定する
`cron`は通常、ログインシェルと環境変数が異なるため、コマンドはフルパスで記載する。

## 2. ログをリダイレクトする
実行結果や実行結果や実行結果やエラーをログファイルに残す。
```
0 * * * * /path/to/script.sh >> /var/log/script.log 2>&1
```

## 3. スクリプトに実行権限を付与する
`chmod +x /path/to/script.sh`を忘れないようにする。

## 4. メール通知の制御
デフォルトでは実行の度にローカルメールが送信される。  
不要な場合は`>/dev/null 2>&1`で捨てるか`MAILTO=""`で設定する。

## 5. 環境変数の設定
スクリプト内で環境変数が必要な場合、`crontab`内で定義するようにする。

## 6. `%`はエスケープする
crontab に設定するコマンドで`%`を使う場合は`\`でエスケープする必要がある。
```
0 8 * * * cd /home/my-user/work && ./example.sh "$(date '+\%Y-\%m-\%d')"
```

# トラブル時にまず確認すること

## 1. 動作確認
`/var/log/syslog`または`/var/log/cron`を確認し、`cron`デーモンが動作しているか、ジョブが実行sれたかを確認する。

## 2. サービス状況
`systemctl status cron`（または`crond`）でサービスが起動しているかを確認する。
