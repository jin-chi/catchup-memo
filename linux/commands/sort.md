# sort
入力値を行ごとに昇順に並べ替えるコマンド。

---

# 基本
```bash
sort input.txt
# 1
# 2
# 3
# ...

# リダイレクトの場合
grep 'error' input.log | sort
```

```bash

# -kオプション
# 区切った中でsortに使用したいフィールドを指定できる
sort -t',' -k1,4 input.txt
-> 上記は1~4カラム目の値を使用してsortする

sort -t',' -k2 input.txt
-> 上記は2カラム目以降のカラムを使用してsortする

sort -t',' -k2,2 input.txt
-> 上記は2カラム目のみを使用してsortする
-> -kオプションでカラムを指定してsortを行うことができる
-> カラムは一つを指定したり複数指定したりすることができる
```

## `-r`オプション
ソートを降順に変更するためのオプション。
```bash
sort -r input.csv
# ...
# 3
# 2
# 1
```

## `-n`オプション
ソートを数値として並べ替える。  
数字だとしても文字列としてソートすると1文字ずつソートするので順序がおかしくなってしまう。  
そのため、数値をソートするときは`-n`を付与することを忘れないこと。
```bash
# -n を指定する
sort -n <<EOF
10
2
100
21
EOF
# -> 2
# -> 10
# -> 21
# -> 100

# 指定しない場合
sort <<EOF
10
2
100
21
EOF
# -> 10
# -> 100
# -> 2
# -> 21
```

## `-t`, `-k`オプション
- `-t`：区切り文字を指定するためのオプション。デフォルトは空白文字（スペース・タブ）。
- `-k`：ソートに使うカラムを指定するためのオプション。
> 列ごとにソートの仕方を指定できるので複雑なソートもできる
```bash
# カンマで区切って、1つ目のカラムのみでソートする
sort -t',' -k1,1 input.txt
```
カラムの指定方法
- `-k2`：2列目以降のカラムでソート
- `-k2,2`：2列目だけでソート
- `-k1,3`：1列目から3列目まででソート
- `-k1,1 -k2,2nr`：1列目でソートした後、2列目を数値として降順でソート

## `-o`オプション
- 出力先ファイルを指定できる
- ソート処理が成功した後にファイルが作成され、出力される
- リダイレクトと違い、エラーになっても空ファイルが作られることがない
- input ファイルと output ファイルが同じ場合にも有効
- リダイレクトだと先にリダイレクトが走るので、読み込む前にファイルが空になってしまう
- 人間が体感できない程度だが、速度はリダイレクトが勝る。しかし大規模データの場合はリダイレクトの方が良い
```bash
# 入力ファイルと同じファイルに出力する場合
sort -o file.txt file.txt

# 入力ファイルと出力ファイルを分ける場合
sort -o output.txt input.txt
```
