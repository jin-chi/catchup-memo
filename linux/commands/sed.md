# sed
正規表現などを使い細かい条件の置換やファイルの編集をする事ができるコマンド。  
入力値を行ごとに処理する。

---

# 基本

## スクリプトコマンド
オプションではない、以下のスクリプトコマンドという機能を持っている。  
これらのコマンドを組み合わせて様々な処理が行える、かなり使用頻度の高いコマンド。
- `g`：一致する全ての箇所を置換 (Global)
- `s`：置換 (Substitute)
- `d`：削除 (Delete)
- `p`：出力 (Print)
- `i`：挿入 (Inesrt)

### `g`：一致する全ての箇所を置換
置換コマンドの最後に`g`をつけることで一致した全ての箇所を置換する。  
`g`をつけない場合は最初に一致した箇所だけ置換する。
```bash
sed 's/aaa/AAA/g' <<< 'aaa bbb aaa ccc'  # -> AAA bbb AAA ccc

# g がないと...
sed 's/aaa/AAA/' <<< 'aaa bbb aaa ccc'  # -> AAA bbb aaa ccc
```

### `s`：置換

```bash
# 最初にヒットした1つだけ置換する
sed 's/aaa/bbb/' input.txt

# ヒットした文字列を全て置換する
sed 's/aaa/bbb/g' input.txt  # g をつける

# / でなくとも特定の記号ならなんでもいい（ただし複数の記号を混ぜないこと）
sed 's|aaa|bbb|g' input.txt  # パイプを使用

# 変数の場合は echo を使用する
echo "${var}" | sed 's/aaa/bbb/g'  # パイプを使用
sed 's/aaa/bbb/g' <(echo "${var}")  # プロセス置換を使用

# リダイレクトも可能
sed 's/aaa/bbb/g' input.txt > output.txt
```

### `d`：削除
```bash
# 2行目を削除
sed '2d' input.txt

# 2行目〜5行目を削除
sed '2,5d' input.txt  # カンマで繋ぐ

# 特定の行から最後まで削除
sed '3,$d' input.txt

# 最終行のみ削除
sed '$d' input.txt

# 特定の文字列を削除
sed '/error/d' input.txt

# 空行を全て削除
sed '/^$/d' input.txt  # 簡単な正規表現は拡張しなくても使える

# 特定の文字で始まる行を削除（この場合はコメントアウト行を削除）
sed '/^#/d' input.txt

# 指定した行以外を削除
sed '1!d' input.txt  # ! をつける

# 複数の特定の行を削除
sed -e '1d' -e '3d' -e '5d' input.txt  # -e オプションを使用する
sed '1d;3d;5d;'  # ; セミコロンを使用する
```
### `p`：出力
`-n`オプション（自動出力の抑制）と一緒に使用するのが基本。  
`-n`オプションを忘れると全行出力される＋ヒットした行がさらに出力される。

```bash
# 2行目を出力
sed -n '2p' input.txt  # 特定の行を出力
sed -n '2,5p' input.txt  # 範囲指定で行を出力
sed -n '3,$p' input.txt  # 特定の行から最終行
sed -n '$p' input.txt  # 最終行のみ

# 特定の文字列にヒットした行を出力
sed -n '/error/p' input.txt  # grep 'error' と同じ
sed -n '/^#/p' input.txt  # 「#」で始まる行

# 一定のパターンで出力
sed -n '1~2p' input.txt  # 奇数行を出力
sed -n '2~2p' input.txt  # 偶数行を出力
sed -n '1~3p' input.txt  # 1行目から3行ごとに出力

# 置換に成功した行を出力
sed -n 's/aaa/bbb/p' input.txt
```

### `i`：挿入
特定の行、またはヒットした行の前に、指定した文字列を挿入する。
```bash
# 一行目に挿入する
sed '1i ----- header -----' input.txt

# 特定の行の前に挿入する
sed '/error/i ----- error -----' input.txt

# 末尾の1つ前に挿入
sed '$i aaa' input.txt  # 末尾に挿入するなら a:Append を使う
```

## -e オプション
実行するスクリプトを指定するためのオプション。  
スクリプトと言っても Linux コマンドではなく、あくまで`sed`で専用の命令のみ。

```bash
sed -e 's/aaa/bbb' input.txt

# 複数の処理を重ねることができる
sed -e 's/aaa/bbb' -e 's/ccc/ddd' input.txt  # 1つ目の処理をしてから2つ目の処理をする

# ; で繋ぐことで同じことができる
sed 's/aaa/bbb; s/ccc/ddd' input.txt  # ; セミコロンで代用可能
```

## -r オプション
`-E`オプションの前の古い書き方。同じ役割を持っている。  
現在は`-E`を使用するのが推奨されている。

```bash
echo "hoge fuga piyo" | sed -r 's/(hoge)/\1_test/'
hoge_test fuga piyo
-> 拡張正規表現を使えるようにするオプション
-> キャプチャグループが()で使用できるようになる
-> -rオプションを使用しなくてもキャプチャグループは使用できるが、()の代わりに$$を使う必要がある。（可読性が下がる）
-> {}や|などがオプションをつけない場合\エスケープが必要になる（こちらも可読性が落ちる）
-> Linuxの環境によっては-rの代わりに-Eを使用する
```

## -E オプション
- 拡張正規表現を使用できるようにするためのオプション
- 拡張正規表現を利用することでより複雑な正規表現を利用する事が可能
```bash
# 特定の文字列に該当する行を置換する
sed -E '/^PASSWORD=/ s/$/pass/' input.conf

sed -E "/^password=/s/$/${PASSWORD}/" input.txt
```

### キャプチャグループ
```bash
echo "hoge fuga piyo" | sed -r 's/(hoge)/\1_test/'
hgoe_test fuga piyo
-> ()で囲んだ値を取得する。\1が取得した内容を表す
```

## -n オプション
- パターンにマッチした行だけ出力する（パターンにマッチしない行は出力しない）
- 特定の値をsedコマンド、正規表現、キャプチャグループで取得する方法
```bash
sed -n 's/.*id=([0-9]+).*/\1/p' input.txt
-> -nとpを組み合わせることで、一致した部分を出力する処理になる
-> ()で囲むことで、一致した部分から()で囲んだ部分を取得できる。()はエスケープが必要。キャプチャグループという。
-> \1はキャプチャグループで取得した値を示している
```

## -i オプション
- 引数に指定したインプットファイル自体を編集するオプション
- デフォルトではファイルを直接編集せずに標準出力されるため、リダイレクトする事が多いが、入力ファイルを直接編集したい場合はこのオプションを指定する
```bash
sed -i 's/hoge/fuga/g' input.txt
```

## -z オプション
区切り文字を`NULL`に変更するオプション。  
デフォルトでは`\n`改行が区切り文字のため、行ごとに処理できているが、`NULL`になることで改行が区切り文字と認識されないため、1行になるという原理。  
しかし、`find ... -print0`などで区切り文字を`NULL`にして値を渡してくる場合は、この機能によって区切り文字を的確に処理することができる。  
1つの値に空白が含まれている場合などに役に立つ。
```bash
sed -z s'/\n//g' input.txt
```

## 正規表現でヒットさせてから処理する
特定の文字列や正規表現でヒットしたる行にのみ処理を行うことができる。
```bash
# pass = の項目にパスワードを追記する
sed "/password =/ s/$/ pass123/" <<EOF
[client]
user = my-user
host = localhost
port = 3306
password =
EOF
# -> [client]
# -> user = my-user
# -> host = localhost
# -> port = 3306
# -> password = pass123

# 実際のファイルだとこうなる
sed -i "/password =/ s/$/ ${PASSWORD}/" example.cnf

# キャプチャグループでもできる
sed -i -E "s/(password =)/\1 ${PASSWORD}/" example.cnf
```

# 応用

## 特定の文字列にヒットした値のみを出力
`-E`, `-n`, `p:出力`を併用することで、ヒットした値のみを出力することができる。  
行全体をヒットさせないと置換する際に漏れてしまうので、左右を`.*`で囲むのがコツ。

```bash
sed -En 's/ID=([0-9]+)/\1/p' <<< 'ID=12345'  # -> 12345

# 左右を「.*」で挟むことで全行ヒットさせる
echo '2023-10-01 [ERROR] Database connection failed' | \
sed -En 's/^([0-9-]+).*\[([^][]*)\].*/\1 \2/p'  # -> 2023-10-01 ERROR
```

## 特定のファイルをまとめてリネームする
`find`, `sed`, `xargs`を組み合わせることで同じパターンのファイル名を一括で変換する。  
`sort`は昇順で出力するために入れているだけなのでマストではない。

```bash
# 以下コメントアウトのコマンドが実行されることになるりファイル名が変更できる
find . -maxdepth 1 -name "*_example*.csv" -type f | sort | sed 'p;s/20241014_//' | xargs -t -n 2 mv

# スペース対策 (-print0, -z, -0 を利用する)
find . -maxdepth 1 -name "*_example*.csv" -type f -print0 | sort -z | sed -z 'p;s/20241014_//' | xargs -0 -t -n 2 mv

# ドライラン (echo を入れる)
find . -maxdepth 1 -name "*_example*.csv" -type f | sort | sed 'p;s/20241014_//' | xargs -n 2 echo mv

# ---

# rename があるならこちらの方がラク
#(ファイル名にスペースが入っている可能性があるなら値をNULL区切りで扱う)
find . -maxdepth 1 -name "*_example*.csv" -type f | xargs rename 's/20241014_//'

# スペース対策 (-print0, -0 を利用する)
find . -maxdepth 1 -name "*_example*.csv" -type f -print0 | xargs -0 rename 's/20241014_//'

# ドライラン (-n オプションをつける)
find . -maxdepth 1 -name "*_example*.csv" -type f -print0 | xargs -0 rename -n 's/20241014_//'
```

## 空白以外の文字列をシングルクオーテーションで囲む
```bash
# []内の^は否定を表す。以下は空白以外という意味。（[]外の^は行頭を表す）
# &はマッチした文字列自体を表す
sed -E "s/[^ ]+/'&'/g" <<EOF
aaa
bbb

ccc
EOF
# -> 'aaa'
# -> 'bbb'
# ->
# -> 'ccc'
```

## CSVの各カラムの値をダブルクォートで囲む
- カンマの後に空白が無いことが前提
- カラムの値にダブルクォートで囲まれたカンマ区切りの文字列があるとバグるので注意
```bash
sed -E 's/[^,]+/"&"/g' input.csv
# "aaa","bbb","ccc"
# "ddd","eee","fff"
# "ggg","hhh","iii"
# ...
```

## 行末のカンマを改行に置換する
```bash
# 行末のカンマを改行に置換
echo 'aaa,bbb,ccc,ddd,' \
sed 's/,$/\n/g'  # aaa,bbb,ccc,ddd <- 末尾に改行

# 単純に削除するだけなら
echo 'aaa,bbb,ccc,ddd,' \
sed 's/,$//g'  # aaa,bbb,ccc,ddd
```

## 最終行の末尾と各行末尾
置換コマンドの外側で指定した値は行を定める。
```bash
# 最終行
sed '$ s/$/,/' input.txt  # の末尾にカンマ

# 最終行以外
sed '$! s/$/,/g' input.txt  # の末尾にカンマ
```
