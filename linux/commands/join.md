# join
  結合するコマンド
- 重複は削除される
- 重複削除は行全体の重複削除はしてくれるが、出力結果を指定した場合は行の内部的に重複削除した後に、出力フィールドを抜き出す流れなので、出力結果は重複が発生することがある
- 結合するファイルはソート必要（結合に使用するキーでソートすること）
- デフォルトの結合キーは一番目のフィールド
- デフォルトの区切り文字はスペースまたはタブ

---

# 基本
- ファイルは指定時にプロセス置換で行うことが多い
- シンプルにファイルを結合する場合は`join`コマンドの方が効率が良い
- 結合時に複雑な処理をする場合は`awk`などのコマンドを使用することを検討する
```bash
join -1 2 -2 4 -o 1.1 1.3 2.2 2.5 <(sort -t, -k2 input1.csv) <(sort -t, -k4 input2.csv) > output.csv
```

## `-j`オプション
- 結合に使用するフィールドを指定するオプション
- デフォルトは一番目のフィールド
```bash
join -j 2 input1.txt input2.txt
```

## `-1`, `-2`, `...`オプション（ファイル番号）
- ファイル番号の後にフィールド番号を指定することで、結合に使用するフィールドをファイルごとに指定することができる
- `-j`の場合は引数のファイルのフィールドをまとめて指定するが、このように個別で指定も可能
```bash
join -1 3 -2 5 input1.txt input2.txt
```

## `-o`オプション
- 出力するフィールドを指定する
- join -o `{ファイル番号}.{フィールド番号}`
- 出力結果を指定すると重複が発生する可能性があるので、必ず結果をユニークにすること
```bash
join -o 1.1 1.2 2.3 input1.txt input2.txt | sort -u
```

## `-t`オプション
- 区切り文字を指定する
```bash
join -t',' -j 2 input1.csv input2.csv
```

## `-v`オプション
- 一致しない行を出力する
- `-v`だけだと両方のファイルの一致しない行が出力される
- ファイル番号も指定すると指定したファイルの行は全て出力される
```bash
# どちらにも存在する行が出力
join -v input1.txt input2.txt

# 指定したファイルの行は全て出力される
join -v 1 input1.txt input2.txt
```
