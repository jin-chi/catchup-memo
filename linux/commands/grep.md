# grep
ファイルや文字列から指定した文字列が含まれる行や値を出力するコマンド。

# zgrep
`gzip`ファイルを`gunzip`しなくても`grep`できる。  
`zgrep`コマンドが内部的に`gunzip`してくれている。  
ほとんどの Linux では`zip`も同じように動作するが、macOS では動作しない。  
また、Linux だとしても必ずできる訳ではないので注意が必要。  
使い方やオプションなどは`grep`と同様。

# 基本
```bash
grep 'ERROR' input.log
```

## -n オプション
該当した行の行数も一緒に表示する。
```bash
grep -n 'ERROR' input.log
```

## -e オプション
OR検索したい場合に使用するオプション。  
`-E`オプションで賄える。
```bash
grep -e 'ERROR' -e 'WARN' input.log

# -E オプションで同じことができる
grep -E "ERROR|WARN" input.log

# 検索文字がハイフンで始まる場合、オプションと勘違いする可能性があるので、その際に使われることがよくある
grep -e '-a' input.log
```

## -E オプション
拡張正規表現を使えるようにするためのオプション。
```bash
grep -E 'ERROR|WARN' input.lot

# egrep と書いても同じ処理だが、grep -E が推奨
grep -E 'ERROR|WARN' input.log
```

## -v オプション
指定した文字列にヒットした行以外を出力する。  
通常は該当したものを出力するが、このオプションは逆に該当したもの以外を出力する。
```bash
grep -v 'INFO' input.log
```

## -q オプション
- grepの結果を出力せずに、ヒットしたかどうかを返す
- 例のように`if`文の条件分岐を使う事が多そう
```bash
if grep -q 'ERROR' input.log; then ...
```

## -o オプション
- ヒットした値の行ごと出力されるのではなく、指定した値だけ取得する事ができる
- `-E`や`-P`などと一緒に正規表現を使用して使われることが多い
```bash
# + を使うために -E を付与
grep -oE 'shopId=[0-9]+' input.log  # -> shopId=123
```

## -h オプション
- 複数ファイルを`grep`するときにどのファイルでヒットしたかわかるようにファイル名が出力される。
- それを抑制したい場合に`-h`オプションが使用される。
```bash
grep -h 'ERROR' input_*.log

# -h がないとファイル名も出力される
# -> input_001.log: [ERROR] Example.log ...
# -> input_002.log: [ERROR] Example.log ...
# -> input_002.log: [ERROR] Example.log ...
# -> input_003.log: [ERROR] Example.log ...
```

# 応用

## -Ff オプション
- 行ごとを検索文字列にする
- ファイルの行ごと検索対象にするときに使う事ができる
- `input1.txt`の行ごとの文字列が`input2.log`に存在するか確認するコマンド。
- `-F`で正規表現が入っていた場合に備えて全て文字列として検索できるようにしている
- `-f`で行ごと検索対象にするようにしている
```bash
grep -Ff input1.txt input2.log

# ハイフンを入れることでリダイレクトの値をファイルとして指定できる
cut -d',' -f3,3 input1.txt | grep -Ff - input2.log  # cut の結果がハイフンに入る
```

## -oP オプション
- `-P`は Perl の正規表現を使えるようにするオプション。`\d`などが使えるようになる
```bash
grep -oP 'shopId=\d+' input.log
# shopId=123
# shopId=456
# shopId=789
```

## ssh との併用
- 実行コマンドはクォーテーションで囲む事
- `ssh`と組み合わせることで別のサーバーに接続して`grep`する事ができる
- 別サーバーにあるログを収集するときに使用した
```bash
ssh example.co.jp "grep 'ERROR' /var/usr/log/input.log"
```

## 文字クラス
- `a-zA-Z`や`0-9`、記号などを表現するための文字クラスが存在する
- 描き以外にも諸々あるので使用する際に調べると良いかも
```bash
grep '[[:alnum:]]' input.log # 英字（大小含む）と数字
grep '[[:alpha:]]' input.log # 英字（大小含む）
grep '[[:upper:]]' input.log # 英字（大文字）
grep '[[:lower:]]' input.log # 英字（小文字）
grep '[[:digit:]]' input.log # 数字
grep '[[:punct:]]' input.log # 記号など
```

## 肯定戻り読み・肯定先読み
- `Perl`の拡張正規表現
- 基本的には`-oP`オプションとセットで使用される
- ヒットした中でさらに特定の部分だけ抽出したい場合に使用する
- 肯定戻り読み：`(?<=)`
  - 左側の肯定的戻り読みは、アンカーをヒット後、戻って左側の文字列がグループキャプチャ内の文字列であるかを確認している
- 肯定先読み：`(?=)`
  - 右側の肯定的先読みは、アンカーをヒット後、右に進んでグループキャプチャ内の文字列であるかを確認している
- 先に出力対象（アンカー）を探して、その後にカッコ内の値とヒットするかを確認する
```bash
grep -oP '(?<=<id>)[^<]\d+(?=</id>)' <<< '<id>123</id>'  # -> 123

# 肯定戻り読みだけでも良い
grep -oP '(?<=ID=)\d+' <<< 'ID=123'  # -> 123

# 肯定戻り読みだけ \K で記載することもできる
grep -oP 'ID=\K\d+' <<< 'ID=123'  # -> 123
```
- 以下のように戻り読み・先読みを使用しないと、指定した文字列全体を出力してしまう
- 上記の例とは異なり、全体が出力対象となるので、タグ内のみを抽出したい場合は他のコマンドを組み合わせるか、上記の方法を使用する必要がある
```bash
grep -oP '<id>[^<]*</id>' <<< '<id>123</id>'  # -> <id>123</id>
```

# アンチパターン

## `-`(ハイフン)で始まる文字検索には注意
以下のような文字列を指定すると、`grep`コマンドがオプションと間違えて予期せぬ挙動になる可能性がある。

```bash
# × : grep には -a オプションがリアルに存在するので特に注意
grep '-a' input.txt

# ○ : -e か -- をつけることで回避できる
grep -e '-a' input.txt
grep -- '-a' input.txt
```
