# pgrep
処理中のプロセスを探すためのコマンド。  
`ps`コマンドと併せて使用する必要がないので便利。

---

# 基本
- デフォルトでは PID（プロセスID）のみを返す
- 他のコマンドへ渡すのに加工が必要なく便利
- 正規表現が使える
- 自分自身の PID 除外する

```bash
pgrep 'java'
```

## `-l`オプション
プロセスIDと一緒にプロセス名も表示するためのオプション。
```bash
pgrep -l 'java'
```

## `-a`オプション
フルコマンドライン（引数など）も含めて表示するためのオプション。
```bash
pgrep -a 'java'
```

## `-u`オプション
特定のユーザーが実行しているプロセスに限定するためのオプション。
```bash
pgrep -u 'user-name'
```

## `-f`オプション
プロセス名だけでなく、コマンドライン全体を検索対象にする。
```bash
pgrep -f 'my-process'
```

---

# 応用

## 自身のプロセスを含まない方法

### 1. 正規表現のカッコを使う（通な方法）
検索したい文字列の最初の1文字を`[ ]`で囲むテクニック。  
`grep`は「`java`」という文字を検索するが、プロセスに表示されるコマンドは「`grep '[j]ava'`」なのでヒットしないという仕組み。
```bash
ps aux | grep '[j]ava'
```

### 2. `grep -v`を繋げる（定番）
シンプルに`grep`を検索対象から除外する方法。
```bash
ps aux | grep 'java' | grep -v 'grep'
```

### 3. `pgrep`と`ps`を組み合わせる（スマート）
`pgrep`で絞り込んだプロセスを`ps`に渡す。  
`ps aux | grep 'java'`と同じ内容を出力できる。
```bash
# pgrep で PID を見つけ、その結果を ps に渡す
ps u -p $(pgrep -f 'java')
```
