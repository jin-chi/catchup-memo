# xargs

標準入力で受け取った値を引数に渡すコマンド。コマンドの処理結果をパイプを通すことで標準入力に変換し、xargsが受け取り次のコマンドの引数にするなどの使い方。

---

# 基本
特徴
1. 改行までを1項目とみなす
2. 改行は空白に変換されて次に渡される
3. 次のコマンドが受け付ける最大の引数の数を超えると、続きの引数から次のコマンドをループする

```bash
# 改行がスペースに変わる
xargs <<EOF
aaa
bbb
ccc
EOF
# -> aaa bbb ccc

# リダイレクトの結果を受け取り、コマンドに渡す
echo 'example.txt' | xargs touch  # -> example.txt ファイルが作成される
```

## `-n`オプション
前の結果を次に渡す際の列数を指定することができる。  
例えば`ls`をした時に`1`を指定すると1列で表示される
```bash
ls | xargs -n 1
# -> example1.txt
# -> example2.txt
# -> example3.txt

xargs -n 1 | awk 'NR>1 { print $1 }' | sort | uniq > work_id.txt

# 2列の場合
ls | xargs -n 2
# -> hoge.txt fuga.txt
# -> piyo.txt
```

## `-I`オプション
- 引数の場所を指定する際に使用するオプション。
- 引数を表すプレースホルダーを指定することで、そのプレースホルダーを指定した場所に引数を配置することができる。
- `-I`を指定することで、区切り文字は「改行」のみに設定される`
```bash
# 全ての CSV を work ディレクトリに移動する
ls *.csv | xargs -I {} mv {} work/
```

## `-0` (`--null`) オプション
- 区切り文字を`NULL`に設定して引数を受け取るためのオプション。（デフォルトはスペース・タブ・改行）
- こうすることで、ファイル名にスペースがある場合でもスペースではなく`NULL`で区切るので想定している処理ができる。
- ただし、パイプを挟んで送り側、受け側の双方で区切り文字の認識を合わせておく必要がある。
- `find`コマンドの場合は、`-print0`で区切り文字を`NULL`に設定している。  
```bash
# find ... -print0 で区切り文字を NULL にして値を渡している
# xargs -0 で区切り文字 NULL に設定することで適切な処理ができる
find . -maxdepth 1 -name '*.txt' -type f -print0 | xargs -0 ./example.sh
```
