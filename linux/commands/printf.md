# printf
Linux の組み込みコマンド。  
`echo`より処理速度も速いのと、環境によって改行が入ったり入らなかったりと予期せぬ挙動になる可能性があるので、ローカルで手軽に使う分には問題ないが、スクリプトなどでツールにしたり本番環境に入れる場合には`printf`を使用した方が良い。  
`printf`は組み込みコマンドと外部コマンドの2種類が存在していることを頭に入れておく。  
ただし、デフォルトでは組み込みコマンドが使用されるため、明示的に指定しない限りは外部コマンドは使用されない。


---

# 基本
```bash
printf '%s\n' "$var"  # デフォルトで改行が入らないので入れるようにする

# 以下の記述方法は変数に % が入った場合にエラーとなるのでしないこと
printf "$var"
```

## -v オプション
出力する代わりに変数に格納するオプション。  
`ksh`, `bash`, `zsh`でしか使えないオプションなので、他のシェルの場合は`bash`などに切り替えて使用する必要がある。

```bash
# 変数 var に代入する
printf -v var 'aaa'  # var=aaa

# フォーマットを指定して代入も可能 (こちらの方がコマンド置換で変数に入れるより軽い)
printf -v var '/home/%s/%s.txt' 'user''example'  # var=/home/user/example.txt

# スクリプトの場合は先頭で宣言する
#-> #!/bin/bash

# bash を指定して実行も可能
bash -c "printf -v var 'aaa'"
```

# 応用

## 数値を0埋めする
```bash
printf '%05d\n' 3  # -> 00003
```

## 3桁ごとのカンマ区切り
```bash
# 整数を3桁ごとのカンマ区切りにする
printf "%'d\n" 123456789  # -> 123,456,789

# 小数点を含む場合
printf "%'f\n" 123456.789  # -> 123,456.789000
```

## 日時の取得
`date`コマンドを使用するより処理が軽くて速い。古い bash(4.2未満)では使えない。  
また、`date`コマンドはナノ秒まで扱えるが、こちらは秒まで。  
ちなみに、どちらもカーネルのシステムクロックの時間を参照しているので、精度は変わらない。
```bash
# %(...)T の構文
printf '%(%Y-%m-%d %H:%M:%S)T\n'  # -> 2026-02-15 20:41:27

# 第二引数を指定する (bash5.0以降は第二引数を省略できる。デフォルトは-1)
printf '%(%Y-%m-%d %H:%M:%S)T\n' -1  # 現在日時
printf '%(%Y-%m-%d %H:%M:%S)T\n' -2  # スクリプト起動日時

# 以下のパラメータで同じ出力ができる
printf '%(%F %T)T\n'  # -> 2026-02-15 20:41:27

# timestamp のフォーマット変更が可能 (ただし timestamp 以外からは date コマンドの方が良い)
printf '%(%F %T)T\n' 1771156037  # -> 2026-02-15 20:47:17
```
